# レシピ詳細タイトル改行とカレンダー選択UIの改善計画

## 背景
- レシピ詳細ヘッダーのタイトルが、モバイル幅でアクションボタン群に押し込まれ、過剰な改行が発生して可読性が低い。
- カレンダー登録モーダルで `input[type="date"]` を利用しており、スマートフォン Chrome でネイティブ日付ピッカーが画面外へ表示されるケースがある。

## 原因の特定
1. **タイトル改行過多の主因**
   - ヘッダー内で「戻る + タイトル + 右側ボタン3個」が同一行に並び、タイトル領域の実効幅が狭い。
   - タイトルの表示行数上限がなく、長い日本語タイトルが4行以上に折り返される。
2. **日付選択UIの画面外表示の主因（モバイル Chrome）**
   - `input[type="date"]` のネイティブピッカー表示位置はブラウザ実装依存。
   - 固定配置モーダル/ボトムシート環境では、端末・Chromeバージョンによってピッカーが視認しづらい位置に出ることがある。
   - 再現条件が端末依存なため、ネイティブ依存を減らすUIへ置換するのが堅実。

## リファクタリング方針
1. **RecipeDetail ヘッダーの可読性改善**
   - タイトルを2行クランプして読みやすさを担保。
   - モバイル時のボタン余白を縮小し、タイトル領域を拡大。
   - `min-w-0` を明示して flex 子要素の縮小挙動を安定化。
2. **CalendarRegistrationModal の日付入力をネイティブ date から分離**
   - 年/月/日の3つの `select` で日付を構築。
   - 月変更時に日付が不正になるケースをクランプ（例: 31日 → 30日）。
   - UI上に整形済み日付プレビューを表示。

## 実施手順
1. `RecipeDetail.tsx` のヘッダークラスとタイトル表示を調整。
2. `CalendarRegistrationModal.tsx` で日付状態の分解・再構成ロジックを追加。
3. `input[type=date]` を `select` 3点構成へ置換。
4. `npm run test` と `npm run build` で回帰確認。
5. モバイル幅でスクリーンショットを取得して見た目確認。

## 期待効果
- 長いタイトルでも視線移動が減り、情報認知がしやすくなる。
- 端末依存のネイティブ日付ピッカー挙動を回避し、スマートフォン Chrome での操作性が安定する。
