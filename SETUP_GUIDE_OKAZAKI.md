## 1. まず確認：設定しんでも使える機能

「設定がえらいめんどくさそうだら…」って思っとるあなたへ。
**ほとんどの機能は、何も設定しんでもすぐ使えるだに！**

| 機能 | 設定なしで使える？ | 必要な設定 |
|------|:-----------------:|-----------|
| レシピ閲覧（約1,700件） | ✅ そのまま使えるじゃん | なし |
| レシピ検索 | ✅ そのまま使えるじゃん | なし |
| 在庫管理（ストック機能） | ✅ そのまま使えるじゃん | なし |
| お気に入り登録 | ✅ そのまま使えるじゃん | なし |
| 閲覧履歴 | ✅ そのまま使えるじゃん | なし |
| 塩分計算 | ✅ そのまま使えるじゃん | なし |
| オフライン利用 | ✅ そのまま使えるじゃん | なし |
| **AIレシピ解析・献立生成** | ❌ 設定いるだに | Gemini APIキー |
| **Googleカレンダーへの登録** | ❌ 設定いるだに | Googleログイン |
| **家族カレンダー連携** | ❌ 設定いるだに | Googleログイン＋カレンダー設定 |
| **複数端末でデータ同期** | ❌ 設定いるだに | Supabase設定 |

まずはそのまま使ってみて、必要んなったら設定すりゃええだに！
焦らんでもええで、な。

## 2. Gemini APIキーの取得と設定

### Gemini APIキーってなんだら？

**Gemini（ジェミナイ）** って言うのは、Googleが作ったAI（人工知能）だに。
Kitchen App では、このAIを使って以下のことができるもんで、設定しとくとえらい便利だよ：

- URLを貼り付けるだけでレシピが自動で読み込まれるだに！
- 在庫にある食材から献立を提案してもらえるじゃん
- 週間献立を自動で作ってくれるだに

**APIキー** ってのは、「あんたがGeminiを使う権利があるで」ってことを証明するパスワードみたいなもんだで。

> ⚠️ **お金のことだら**: Gemini APIには無料枠があるもんで安心しりん（2025年現在、個人で使う分なら無料枠で全然足りるだに）。クレジットカードなしでも無料枠の分は使えるじゃん。

---

### ステップ 1: Googleアカウントを用意するだに

GmailやYouTubeで使っとるGoogleアカウントでええだに。
持っとらない場合は [accounts.google.com](https://accounts.google.com) から作っておくりん。

---

### ステップ 2: Google AI Studio に行くだに

1. ブラウザで **[https://aistudio.google.com](https://aistudio.google.com)** を開くだに
2. 右上の「**Sign in**」か「**ログイン**」をタップするじゃん
3. お使いのGoogleアカウントでログインしりん

---

### ステップ 3: APIキーを作るだに

1. 左側のメニューから「**Get API key**」（APIキーを取得）をクリックするだに
   ※ メニューが見当たらん場合は、左上のハンバーガーメニュー（三本線）をタップしりん

2. 「**Create API key**」（APIキーを作成）ボタンをクリックするじゃん

3. 「**Create API key in new project**」を選んでクリックするだに
   （「新しいプロジェクトにAPIキーを作る」って意味だで）

4. ちょっと待つと、英数字の長い文字列（例: `AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`）が出てくるじゃん

5. 「**コピー**」ボタンをクリックしてAPIキーをコピーしりん
   ⚠️ **このキーは他の人に見せたらいかんで！**

---

### ステップ 4: アプリにAPIキーを入れるだに

1. Kitchen App を開くじゃん
2. 画面下のナビゲーションバーから「**設定**」（歯車アイコン）をタップするだに
3. 「**Gemini APIキー**」って項目を探してちょ
4. 鍵アイコンをタップしてロックを解除するだに
5. テキストボックスにコピーしたAPIキーを貼り付けるじゃん
6. 「**保存**」ボタンをタップするだに
7. 「**接続テスト**」ボタンをタップして「接続成功」って出てくればバッチリだで！

---

### ステップ 5: ちゃんと動くか確かめりん

1. 画面下のナビゲーションから「**AI解析**」を開くだに
2. 好きなレシピサイトのURLを貼り付けてみりん
3. レシピが自動で読み込まれりゃ設定完了だじゃん！やったね！

## 3. Googleログインのやり方

### なんでGoogleログインが必要だら？

GoogleカレンダーにレシピやPayカレンダーに献立を登録したり、家族とスケジュールを共有するには、
アプリが「あんたのGoogleカレンダーにアクセスする許可」をもらわないかんだもんで。
その許可をもらうのが「Googleログイン」ってやつだに。

> ℹ️ Googleログインには **Supabase**（スーパーベース）ってサービスの設定も必要だで。
> Supabaseってのは、アプリのデータを管理するためのサービスだに。[第5章](#5-supabaseのセットアップクラウド同期)を先に読んでから戻ってきりん。

---

### やる前に確認しとくこと

- Supabaseのセットアップが終わっとること（第5章参照だで）
- Google Cloud Console でOAuthの設定が終わっとること（以下で説明するじゃん）

---

### ステップ 1: Google Cloud Console に行くだに

**Google Cloud Console（グーグル・クラウド・コンソール）** ってのは、GoogleのAPIを使うための管理画面だに。
えらい難しそうに見えるけど、順番通りにやりゃ全然大丈夫だで！

1. ブラウザで **[https://console.cloud.google.com](https://console.cloud.google.com)** を開くじゃん
2. Googleアカウントでログインするだに
3. 利用規約が出てきたら確認して「**同意する**」をクリックしりん

---

### ステップ 2: プロジェクトを作るだに

**プロジェクト** ってのは、アプリの設定をまとめる「フォルダ」みたいなもんだで。

1. 画面上部の「**プロジェクトを選択**」をクリックするじゃん
2. 右上の「**新しいプロジェクト**」をクリックするだに
3. 以下を入力してちょ：
   - プロジェクト名: `Kitchen App`（なんでもええだに）
4. 「**作成**」をクリックするじゃん
5. 作成されたプロジェクトを選択するだに

---

### ステップ 3: Google Calendar API を有効にするだに

1. 左側のメニューから「**APIとサービス**」→「**ライブラリ**」をクリックするじゃん
2. 検索ボックスに「**Google Calendar API**」って入れて検索するだに
3. 「Google Calendar API」をクリックしりん
4. 「**有効にする**」ボタンをクリックするじゃん
5. 「APIが有効になりました」って出てくればOKだで！

---

### ステップ 4: OAuth同意画面を設定するだに

**OAuth同意画面** ってのは、「このアプリはあんたのGoogleカレンダーにアクセスしたいだに」って確認画面のことだで。

1. 左側メニューから「**APIとサービス**」→「**OAuth同意画面**」をクリックするじゃん
2. 「**外部**」を選んで「**作成**」をクリックするだに
3. アプリ情報を入力してちょ：
   - アプリ名: `Kitchen App`
   - ユーザーサポートメール: 自分のGmailアドレス
   - デベロッパーの連絡先: 自分のGmailアドレス
4. 「**保存して次へ**」をクリックするじゃん
5. スコープの画面では「**保存して次へ**」をクリックするだに（そのままでええだで）
6. テストユーザーの画面で「**+ ADD USERS**」をクリックして、自分のGmailアドレスを追加しりん
7. 「**保存して次へ**」→「**ダッシュボードに戻る**」をクリックするじゃん

---

### ステップ 5: OAuth 2.0 クライアントIDを作るだに

1. 左側メニューから「**APIとサービス**」→「**認証情報**」をクリックするじゃん
2. 上部の「**+ 認証情報を作成**」→「**OAuth クライアント ID**」をクリックするだに
3. アプリケーションの種類: 「**ウェブ アプリケーション**」を選ぶじゃん
4. 名前: `Kitchen App Web Client`（なんでもええだに）
5. 「**承認済みのリダイレクト URI**」に以下を追加してちょ：
   ```
   https://あなたのプロジェクトID.supabase.co/auth/v1/callback
   ```
   ※ `あなたのプロジェクトID` はSupabaseのプロジェクトIDに置き換えりん
6. 「**作成**」をクリックするじゃん
7. **クライアントID** と **クライアントシークレット** が出てくるもんで → メモしとくだに！

---

### ステップ 6: SupabaseにGoogleプロバイダーを設定するだに

1. [app.supabase.com](https://app.supabase.com) を開いてログインするじゃん
2. 自分のプロジェクトを選ぶだに
3. 左側メニューから「**Authentication**」→「**Providers**」をクリックするじゃん
4. 「**Google**」を見つけて「**Enable**」をオンにするだに
5. 以下を入力してちょ：
   - **Client ID（クライアントID）**: ステップ5でメモしたやつ
   - **Client Secret（クライアントシークレット）**: ステップ5でメモしたやつ
6. 「**Save**」をクリックするじゃん

---

### ステップ 7: アプリでGoogleログインをするだに

1. Kitchen App を開くじゃん
2. 「**設定**」画面を開くだに
3. 画面上のセクション一覧から「**アカウント**」を選ぶだに
4. 「**Googleでログイン**」ボタンをタップするじゃん
5. Googleのログイン画面（`accounts.google.com`）が開くだに
6. 自分のGoogleアカウントを選ぶじゃん
7. 「カレンダーへのアクセスを許可しますか？」って確認が出てきたら「**許可**」をタップするだに
8. アプリに戻ってログイン済みの表示になればバッチリだじゃん！

> 補足: 今はホーム右上の「ログイン」ボタンはなくなっとるだに。ログインは設定画面の「アカウント」からやってちょ。

## 4. 家族カレンダーの設定

### これを設定するとどえらい便利だに！

- 今日の献立をGoogleカレンダーに自動で登録できるじゃん
- 家族のスケジュールを見て、外食の日を避けた献立を提案してくれるだに
- 食事時間もカスタマイズできるじゃん（例：夕食は18:00〜19:00）

> 前提: [第3章のGoogleログイン](#3-googleログインのやり方)が終わっとること

---

### ステップ 1: Googleカレンダーで家族用カレンダーを作るだに

1. パソコンかスマートフォンで **[calendar.google.com](https://calendar.google.com)** を開くじゃん
2. 左側の「**他のカレンダー**」の横にある「**+**」ボタンをクリックするだに
3. 「**新しいカレンダーを作成**」をクリックするじゃん
4. 以下を入力してちょ：
   - 名前: `家族の予定`（なんでもええだで）
   - 説明: 任意
5. 「**カレンダーを作成**」をクリックするだに

---

### ステップ 2: カレンダーIDを調べるだに

**カレンダーID** ってのは、それぞれのカレンダーを区別するための番号（ID）だで。
メールアドレスみたいな見た目してるじゃん（例: `xxxx@group.calendar.google.com`）。

1. Googleカレンダーの左側に、作ったカレンダーが表示されとるじゃん
2. カレンダー名の右にある「**⋮（三点メニュー）**」をクリックするだに
3. 「**設定と共有**」をクリックするじゃん
4. 下にスクロールして「**カレンダーの統合**」って項目を探してちょ
5. 「**カレンダーID**」って欄に書かれた文字列をコピーするだに
   例: `abcdefg1234@group.calendar.google.com`

---

### ステップ 3: アプリにカレンダーIDを設定するだに

1. Kitchen App の「**設定**」を開くじゃん
2. 「**カレンダー設定**」って項目を探してちょ
3. 以下を設定するだに：

| 設定項目 | 入力内容 | 説明 |
|---------|---------|------|
| 献立登録先カレンダー | コピーしたカレンダーID | 献立を登録するカレンダーだで |
| 家族カレンダー | コピーしたカレンダーID | 家族の予定を参照するカレンダーだに |
| 食事開始時間 | 例: `18:00` | 夕食の開始時間じゃん |
| 食事終了時間 | 例: `19:00` | 夕食の終了時間だで |

4. 「**保存**」をタップするじゃん

---

### ステップ 4: ちゃんと動くか確かめりん

1. 「**週間献立**」画面を開くだに
2. 「**カレンダーに登録**」ボタンをタップするじゃん
3. Googleカレンダーを開いて、献立が登録されとりゃOKだで！

---

### 家族カレンダーとの連携について

家族カレンダーに「外食」「旅行」みたいな予定が入っとる日は、
AIが献立を提案するときにその日を考慮してくれるだに。
うまく使って、毎日の料理をもっとラクにしてちょ！

## 5. Supabaseのセットアップ（クラウド同期）

### Supabaseってなんだら？

**Supabase（スーパーベース）** ってのは、データをインターネット上に保存・管理するためのサービスだで。
料理に例えると、「冷蔵庫（スマホ本体）」のデータを「コンビニ（クラウド）」にもバックアップしとくイメージだに。

#### Supabaseを設定するとできることだに：
- 複数のスマートフォンやパソコンでデータを同期できるじゃん
- スマホが壊れてもデータが消えんで済むだに！
- 家族でレシピや在庫を共有できるじゃん

#### 設定しんでもできること：
- 全部の基本機能はスマホの中だけで動くだに（インターネット不要だで）

> ⚠️ **お金のことだら**: Supabaseには無料プランがあるもんで安心しりん（2025年現在、個人利用なら無料枠で全然OK）

---

### ステップ 1: Supabaseのアカウントを作るだに

1. ブラウザで **[https://supabase.com](https://supabase.com)** を開くじゃん
2. 右上の「**Start your project**」か「**Sign Up**」をクリックするだに
3. 「**Continue with GitHub**」（GitHubアカウントで登録）か「**Sign up with email**」（メールで登録）を選ぶじゃん
   - GitHubアカウントを持っとる場合はGitHubでの登録がラクだで
   - メール登録の場合はメールアドレスとパスワードを入力するだに
4. 確認メールが届いたらリンクをクリックするじゃん（メール登録の場合）

---

### ステップ 2: プロジェクトを作るだに

1. ログイン後、「**New Project**」（新しいプロジェクト）をクリックするじゃん
2. 以下を入力してちょ：
   - 組織（Organization）: 「**New organization**」って表示されとりゃそのままでええだに
   - プロジェクト名: `kitchen-app`（英語で入力するだで）
   - データベースパスワード: 長くて覚えやすいパスワード（英数字混在がええだに）
     ⚠️ 絶対メモしといてちょ！後から変えるのがえらい大変だもんで
   - リージョン（Region）: `Northeast Asia (Tokyo)` を選ぶじゃん（日本に近い方が速いだで）
3. 「**Create new project**」をクリックするだに
4. 1〜2分待つとプロジェクトが作られるじゃん

---

### ステップ 3: APIキーとURLを確認するだに

1. プロジェクトのダッシュボードが開いたら、左側メニューの「**Project Settings**」をクリックするじゃん
2. 「**API**」をクリックするだに
3. 以下の2つをメモかコピーしといてちょ：
   - **Project URL**: `https://xxxxxxxxxxxx.supabase.co` って形式のやつだで
   - **anon public key**: 長い文字列（`eyJhbGci...`で始まるやつだに）

---

### ステップ 4: データベースのテーブルを作るだに

アプリのデータを保存するための「テーブル（表）」をデータベースに作るじゃん。

1. 左側メニューから「**SQL Editor**」をクリックするだに
2. 「**+ New query**」をクリックするじゃん
3. プロジェクト内の `supabase/migrations/001_initial_schema.sql` ファイルの中身をコピーするだに
4. SQLエディタに貼り付けて「**Run**」（▶ ボタン）をクリックするじゃん
5. 「**Success**」って表示されればOKだで！
6. 同じように `supabase/migrations/002_rls_policies.sql` もやってちょ

---

### ステップ 5: アプリに設定を入れるだに

プロジェクトのルートフォルダに `.env` ってファイルを作るじゃん。

1. `.env.example` ファイルをコピーして `.env` って名前にするだに
   （ターミナルで `cp .env.example .env` って入力してもええだで）

2. `.env` ファイルをテキストエディタで開いて、以下を入力してちょ：

```
VITE_SUPABASE_URL=https://あなたのプロジェクトID.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGci...（anon publicキー）
```

3. ファイルを保存するじゃん

4. アプリを再起動するだに（`npm run dev` をもっかい実行するじゃん）

---

### ステップ 6: ちゃんと動くか確かめりん

1. Kitchen App を開くだに
2. 「**設定**」画面を開くじゃん
3. 「**Googleでログイン**」でログインするだに
4. ログインできたら、データがクラウドと同期されるようになるじゃん！やったね！

## 6. アプリをホーム画面に追加するやつ（PWAインストール）

Kitchen App は **PWA（プログレッシブ・ウェブ・アプリ）** に対応しとるだに。
ホーム画面に追加しとくと、普通のアプリみたいに起動できて、えらい便利だじゃん！

### iPhoneの場合（Safariじゃないといかんで）

> ⚠️ **必ずSafariで開いてちょ。Chromeや Firefoxでは追加できんだに！**

1. **Safari** でアプリのURLを開くじゃん
2. 画面下部の「**共有ボタン**」（四角から矢印が出とるアイコン）をタップするだに
3. メニューを下にスクロールして「**ホーム画面に追加**」をタップするじゃん
4. アプリ名（「Kitchen App」）を確認して「**追加**」をタップするだに
5. ホーム画面にアイコンが追加されるじゃん！

---

### Androidの場合（Chrome）

1. **Chrome** でアプリのURLを開くだに
2. 画面の上か下に「**アプリをインストール**」ってバナーが出てくることがあるじゃん
   出てきたらそれをタップするだに
3. バナーが出てこん場合：
   - Chromeの右上「**⋮（三点メニュー）**」をタップするじゃん
   - 「**アプリをインストール**」か「**ホーム画面に追加**」をタップするだに
4. 「**インストール**」をタップするじゃん
5. ホーム画面にアイコンが追加されるだに！

---

### オフライン利用についてだに

一回アプリを開いてデータを読み込んだ後は、
インターネットに繋がっとらんくてもレシピの閲覧・在庫管理なんかができるだじゃん！
（AI機能やカレンダー連携はオンラインじゃないといかんだで）

## 7. 困ったときはここ見りん

### Q: Gemini APIキーを入力したのに「接続失敗」って出てくるだら

**ここを確認してちょ：**
1. キーの先頭・末尾にスペースが入っとらんか確認するじゃん
2. Google AI Studio で作ったキーがちゃんとコピーできとるか確認するだに（全部で39文字前後だで）
3. Google AI Studio の無料枠の上限に達しとらんか確認するじゃん（[aistudio.google.com](https://aistudio.google.com) でステータスを確認してちょ）

### Q: Googleログインができん・エラーが出るだら

**ここを確認してちょ：**
1. 設定画面の「アカウント」に「Googleでログイン」ボタンが出とるか確認するじゃん
2. `VITE_GOOGLE_CLIENT_ID` が環境変数に設定されとるか確認するだに（未設定だとログインボタンは無効になるじゃん）
3. Supabaseの設定でGoogleプロバイダーが「有効」になっとるか確認するじゃん
4. Google Cloud Console でOAuth同意画面の「テストユーザー」に自分のGmailを追加したか確認するだに
5. リダイレクトURIがちゃんと入力されとるか確認するじゃん（スラッシュのあるなしに注意してちょ）
6. アプリ内ブラウザじゃなくて、普通のChrome/Safariで開いて試すだに（ポップアップ制限対策）

> `https://accounts.google.com` を直接開くことはできるけど、これはGoogleアカウントに入るだけだに。Kitchen App の連携まで終わらせるには、アプリの「Googleでログイン」ボタンから始める必要があるじゃん。

### Q: アプリのデータが消えてしまっただら

データはスマホ本体に保存されとるだに。以下の操作をするとデータが消えてしまうじゃん：

- ブラウザのキャッシュをぜんぶ削除した
- プライベートブラウジング（シークレットモード）で使っとった
- アプリをアンインストールした

**対策だに:** [第5章のSupabase設定](#5-supabaseのセットアップクラウド同期)をやっとくと、クラウドにバックアップが作られて安心だじゃん！

### Q: `.env` ファイルはどこに作りゃええだら

アプリのフォルダの一番上（ルートディレクトリ）に作るだに。
`package.json` と同じ場所に置きゃOKだじゃん。

```
my-recipe-app/
├── .env          ← ここに作るだに！
├── .env.example  ← これをコピーして使うじゃん
├── package.json
├── src/
└── ...
```
