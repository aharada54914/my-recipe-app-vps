# 個人設定のしかた（初期設定ガイド）

Kitchen App をもっと便利にもっと安全に使うための「３つのお守り設定」についてご紹介します。難しそうに見えても大丈夫、一つずつ順番に進めれば誰でも設定できます！

---

## 1. Google アカウント連携（データのお守り）

「スマホを買い替えたらレシピがごそっと消えちゃった…」という悲しい事故を防ぐための、いちばん大切な設定です。あなたがお使いの Google ドライブの隅っこに、アプリのデータをまるごと自動で保存（バックアップ）してくれます。

### どうやって設定するの？
1. アプリの **「設定」 > 「アカウント」** を開きます。
2. もし `VITE_GOOGLE_CLIENT_ID` という環境変数が設定されていれば、そこに **「Googleでログイン」** というボタンが表示されます。（※表示されていない場合は、アプリの配布元やご家族の管理者に「Googleログインを有効にしてほしい」とお伝えください。）
3. ボタンを押して、普段お使いの Google アカウントを選ぶだけ！
4. これで完了です。レシピを追加したり献立を変えたりするたびに、自動で安全にバックアップされるようになります。

---

## 2. Google カレンダー連携（家族で共有するためのお守り）

Kitchen App で作った「今週の献立表」や「お買い物リスト」を、スマートフォンのカレンダーアプリに予定として書き込むことができます。ご家族とカレンダーを共有していれば、「今日の夜ご飯なに？」と聞かれる前に自動でお知らせできます！

### どうやって設定するの？
1. まず上記の **Google アカウント連携** を終わらせておきます。
2. アプリの **「設定」 > 「カレンダー」** を開きます。
3. カレンダーへの書き込み許可（スイッチ）をオンにします。
4. **「どのカレンダーに書き込むか」** を選びます（「家族用カレンダー」などの共有カレンダーを選ぶのがおすすめです）。
5. あとは、週間献立の画面で「カレンダー登録」ボタンを押すだけ！ 指定したカレンダーに献立予定と買い物リストがバッチリ登録されます。

---

## 3. Gemini API キー（AIのちからを借りるためのお守り）

「冷蔵庫の残り物から今日の献立を考えてほしい…」「ネットで見つけたレシピを全自動で読み込んでほしい…」そんなときに活躍するのが最新のAI機能です。あなた専用の「合鍵（APIキー）」を取得してアプリに教えることで、AIがフルパワーを発揮してくれます。

### どうやって設定するの？
1. Google の AI向けサイト [Google AI Studio](https://aistudio.google.com/app/apikey) にアクセスします（Googleアカウントでログインしてください）。
2. **「Create API key」**（APIキーを作成）というボタンを探して押します。
3. `AIza...` から始まる長いパスワードのような文字列が表示されたら、それを見失わないようにコピーします。これが「APIキー」です！
4. Kitchen App の **「設定」 > 「献立」** タブを開きます。
5. **「Gemini API（AI連携機能）の設定」** という項目の横にある「ロック中」ボタンを押してロックを解除します。
6. 先ほどコピーした長い文字列を貼り付けて、「保存」を押します。
7. 「接続テスト」ボタンを押して「✅ 接続成功」と表示されれば、設定完了です！AIがいつでもお手伝いできる状態になりました。

### よくある質問
* **Q. この設定はお金がかかるの？**
  * A. 基本的にすべて無料の範囲でご利用いただけます。AI（Gemini API）も通常のご家庭での利用範囲なら無料枠で十分にまかなえます。
* **Q. APIキーを誰かに知られても平気？**
  * A. **絶対に秘密にしてください。** 銀行の暗証番号と同じくらい大切なものです。Kitchen App はあなたのスマホの中（ローカル）にだけこのキーを保存し、こっそり外部に漏らすようなことは一切ありませんのでご安心ください。

## 利用者側と開発者側で「やること」の違い

同じ Google 連携でも、**利用者（アプリを使う人）** と **開発者（アプリを配布・設定する人）** で、必要な作業が違います。

### 利用者側（アプリを使う人）が必要なこと
1. アプリの **「設定 > アカウント」** で **「Googleでログイン」** を実行する。  
   （ボタンがある場合は、開発者側の準備は完了しています）
2. バックアップを使う場合は、そのまま使えばOK（自動で Google Drive に保存されます）。
3. カレンダー連携を使う場合は **「設定 > カレンダー」** で連携をONにし、書き込み先カレンダーを選ぶ。

### 開発者側（アプリを配布・設定する人）が必要なこと

#### Step 1: Google Cloud の準備
1. [Google Cloud Console](https://console.cloud.google.com/) にログインする。
2. 画面上部のプロジェクト選択から **「新しいプロジェクト」** を作成する。
3. 請求先設定が必要と表示される場合は、案内に従って設定する（無料枠内でも設定が必要な場合があります）。

#### Step 2: 必要な API を有効化
1. 左メニューの **「APIとサービス」 > 「ライブラリ」** を開く。
2. **Google Calendar API** を検索して「有効にする」を押す。
3. 同じように **Google Drive API** も有効にする。

#### Step 3: OAuth 同意画面を作る
1. **「APIとサービス」 > 「OAuth 同意画面」** を開く。
2. ユーザータイプを **External** にする（家族や友人など外部ユーザーが使うため）。
3. アプリ名・サポートメール・連絡先メールを入力して保存する。
4. テスト公開中の場合は、必要に応じてテストユーザー（自分のGoogleアカウント等）を追加する。

#### Step 4: OAuth Client ID（Web）を作る
1. **「APIとサービス」 > 「認証情報」 > 「認証情報を作成」 > 「OAuth クライアント ID」** を選ぶ。
2. アプリの種類は **「ウェブアプリケーション」** を選択する。
3. 名前を付けて作成する（例: `kitchen-app-web`）。

#### Step 5: JavaScript origin を登録する
1. 作成した OAuth Client の編集画面を開く。
2. **Authorized JavaScript origins** に次を登録する。
   - `http://localhost:5173`（ローカル開発用）
   - 本番URL（例: `https://your-app.vercel.app`）
3. 保存して、表示された **Client ID** をコピーする。

#### Step 6: 環境変数を設定して反映する
1. プロジェクトの `.env` に次を追加する。
   - `VITE_GOOGLE_CLIENT_ID=コピーしたClient ID`
2. 開発サーバーを再起動する（`npm run dev` をやり直す）。
3. 設定画面で **「Googleでログイン」** ボタンが表示されることを確認する。

#### Step 7: 本番環境（Vercelなど）にも同じ値を入れる
1. ホスティング先の環境変数設定画面を開く。
2. `VITE_GOOGLE_CLIENT_ID` を本番環境にも登録する。
3. 再デプロイして、本番URLでもログインできることを確認する。

### つまずきポイント（切り分けのコツ）
- **利用者側の画面に「Googleでログイン」が出ない**: 開発者側の `VITE_GOOGLE_CLIENT_ID` 設定が未完了の可能性が高い。
- **ログインはできるがカレンダー登録が失敗する**: 開発者側で Calendar API の有効化・OAuth設定を再確認する。
- **バックアップが動かない**: 開発者側で Drive API の有効化を再確認する。
