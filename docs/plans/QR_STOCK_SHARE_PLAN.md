# 在庫QR共有 実装プラン（MVP）

最終改訂: 2026-02-25

## 目的

近距離で在庫データを共有するため、QRコードを使って在庫情報を安全に受け渡しできるようにする。

## 前提

- 既存のエクスポート/インポート基盤（overwrite/merge）を活用する。
- 全データではなく、まずは `stock` テーブルのみを対象にする。
- 大容量データは分割QR（連番付き）で扱う。

## スコープ（MVP）

### 送信側

1. 在庫データを取得（`db.stock.toArray()`）
2. JSON化
3. 圧縮（deflate など）
4. Base64URL化
5. チャンク分割（例: 700〜900文字）
6. 連番メタ付きQRを順番表示

QRペイロード例:

```json
{
  "v": 1,
  "session": "8char",
  "index": 1,
  "total": 6,
  "crc32": "...",
  "payload": "..."
}
```

### 受信側

1. カメラで連続スキャン
2. `session` ごとにチャンク収集
3. `total` 分揃ったら順序復元
4. CRC検証
5. 解凍・JSON復元
6. `stock` データとして取り込み（merge / overwrite 選択）

## UI案

- 設定 > データ管理 に以下を追加
  - 「在庫をQRで共有（送信）」
  - 「在庫をQRで受け取る（受信）」
- 受信画面では進捗を表示
  - 受信済み枚数（`3/6`）
  - 不足番号（`#4 未受信`）

## エラーハンドリング

- セッションID不一致は破棄
- CRC不一致は再スキャン案内
- タイムアウト時に再開/破棄を選択
- カメラ権限拒否時は「コード貼り付け」フォールバックを表示

## 技術メモ

- QR生成: `qrcode` 系ライブラリを検討
- QR読み取り: `html5-qrcode` / `@zxing/browser` を比較
- PWA（iOS Safari）でのカメラ権限制約を実機確認必須

## 段階リリース

### Phase 1

- 在庫のみ
- 分割QR送受信
- merge取り込みのみ

### Phase 2

- overwrite対応
- 受信中断からの再開
- 文字列コード手入力/貼り付け

### Phase 3

- 共有対象の拡張（favorites, notes など）
- 署名 or PIN で誤取り込み防止
