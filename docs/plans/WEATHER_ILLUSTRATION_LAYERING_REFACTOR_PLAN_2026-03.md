# 天気イラスト重ね合わせ表現 リファクタリング計画（2026-03）

## 1. 目的


## 0.1 追加要件の確定（今回反映）

- 湿度情報: 取得可能な場合は表示に含める（JMAレスポンスの `humiditys` を優先利用）。
- イラストトーン: **フラット寄り** を採用。
- 夜間表現: **月アイコン連動を有効化**。
- 雷・雪: **安全性重視で強調色を強める**（注意喚起優先）。

取得済みの週間天気データ（気象庁API）を、
単なる数値表示（気温/降水）ではなく、
**各日の天候を直感的に理解できるイラスト表現**へ改善する。

要件:
- 検知した天気関連単語（例: 晴れ/曇り/雨/雷/雪/霧/強風）から、
  レイヤー合成でイラストを生成する。
- 表示は高速（リスト描画に耐える）で、見た目の統一感がある。
- 週間献立ページの天気カードで利用可能。

---

## 2. 現状とギャップ

### 現状
- 週間天気は気象庁APIから取得し、日付・気温・降水目安をテキスト表示している。
- 天候の意味（体感/印象）は数字を読まないと伝わりづらい。

### ギャップ
1. 天候カテゴリ判定（晴/曇/雨…）がUI表現と分離されていない。
2. イラスト資産管理方針（ベース + オーバーレイ）が未定義。
3. 表示性能（毎週7日×再描画）向けのキャッシュ戦略が未実装。
4. デザイン一貫性（色・サイズ・重なり優先度）が仕様化されていない。

---

## 3. 方針（アーキテクチャ）

### 3.1 3層分離
1. **Weather NLP/Rule 層**
   - 取得データ/天気文言/コードからトークン抽出。
   - `sunny`, `cloudy`, `rain`, `snow`, `storm`, `windy`, `fog` など正規化タグへ変換。
2. **Illustration Compose 層**
   - ベースアイコン + オーバーレイ（雨粒、雷、雪、霧、風線）を重ね合わせ。
   - z-index順と不透明度ルールで合成。
3. **UI Presentation 層**
   - 合成結果を `WeatherIllustration` コンポーネントで表示。
   - 天気カード/将来のホーム画面にも再利用。

### 3.2 新規モジュール案
- `src/utils/season-weather/weatherTagger.ts`
  - 天気文言/コード→タグ配列
- `src/utils/season-weather/weatherIllustrationComposer.ts`
  - タグ→レイヤー構成
- `src/components/weather/WeatherIllustration.tsx`
  - SVG/画像レイヤーのレンダリング
- `src/components/weather/weatherIllustrationTokens.ts`
  - カラーパレット・サイズ・重なり定数

### 3.3 アセット戦略
- まずは**SVGパーツ合成**を採用（軽量・色調整容易）。
- ベース層: 空/太陽/月/雲
- 効果層: 雨・雪・雷・霧・風
- アクセント層: 時間帯トーン（朝/昼/夜）

---

## 4. 単語検知と重ね合わせルール

### 4.1 入力ソース
- 優先1: 気象庁天気コード
- 優先2: 気象庁の天気テキスト
- 優先3: 降水量・気温・湿度閾値から補完タグ

### 4.2 タグ化ルール（例）
- `晴` を含む → `sunny`
- `曇` を含む → `cloudy`
- `雨` を含む or 降水量しきい値超え → `rain`
- `雪` を含む or 低温 + 降水有 → `snow`
- `雷` を含む → `storm`
- `霧` を含む → `fog`
- `風`/`強風` を含む → `windy`

### 4.3 合成優先順位（z-index）
1. sky gradient
2. sun/moon
3. cloud base
4. precipitation (rain/snow)
5. special effect (storm/fog/wind)
6. highlight (glow/spark)

### 4.4 センスの良い見た目の指針
- 同時表示最大3効果まで（情報過多回避）
- 彩度を気温帯で微調整（暑い日=暖色寄り、寒い日=寒色寄り）
- 雨/雪粒子は密度を降水強度で段階制御
- 「曇り時々雨」等は主タグ1+副タグ1で簡潔化
- 夜間帯は月アイコンを基本表示（晴れ/薄曇で優先）
- 雷・雪は警戒色（高彩度イエロー/シアン）を強めに適用

---

## 5. パフォーマンス設計

1. タグ判定を `useMemo` / pure function 化。
2. 日別 `weatherKey`（`date+code+temp+precip`）で合成結果をキャッシュ。
3. SVGパーツをシンボル化し再利用。
4. 一覧描画は `React.memo(WeatherIllustration)` で無駄再描画を抑制。

目標:
- 7日分初回描画: 16ms/frameを維持
- 週切替時の体感遅延: 100ms未満

## 8. 受け入れ基準（DoD）

1. 各日の天気がイラストで直感表示される。
2. 単語検知（またはコード）から適切なレイヤー合成が行われる。
3. 週間献立ページで折りたたみ展開時に崩れない。
4. 低速端末でも実用性能を維持する。
5. API失敗時はテキスト表示へ安全にフォールバックする。
