## 1. まず確認：設定しなくても使える機能

「設定が難しそう…」と思ったあなたへ。
**多くの機能は、何も設定しなくてもすぐに使えます！**

| 機能 | 設定なしで使える？ | 必要な設定 |
|------|:-----------------:|-----------|
| レシピ閲覧（約1,700件） | ✅ そのまま使える | なし |
| レシピ検索 | ✅ そのまま使える | なし |
| 在庫管理（ストック機能） | ✅ そのまま使える | なし |
| お気に入り登録 | ✅ そのまま使える | なし |
| 閲覧履歴 | ✅ そのまま使える | なし |
| 塩分計算 | ✅ そのまま使える | なし |
| オフライン利用 | ✅ そのまま使える | なし |
| **AIレシピ解析・献立生成** | ❌ 要設定 | Gemini APIキー |
| **Googleカレンダーへの登録** | ❌ 要設定 | Googleログイン |
| **家族カレンダー連携** | ❌ 要設定 | Googleログイン＋カレンダー設定 |
| **複数端末でデータ同期** | ❌ 要設定 | Supabase設定 |

まずはそのまま使ってみて、必要になったら設定していきましょう！

## 2. Gemini APIキーの取得と設定

### Gemini APIキーとは？

**Gemini（ジェミナイ）** は、Google が作ったAI（人工知能）です。
Kitchen App では、このAIを使って以下のことができます：

- URLを貼り付けるだけでレシピを自動で読み込む
- 在庫にある食材から献立を提案してもらう
- 週間献立を自動生成する

**APIキー** とは、「あなたがGeminiを使う権利がある」ことを証明するパスワードのようなものです。

> ⚠️ **費用について**: Gemini APIには無料枠があります（2025年現在、個人利用なら通常無料枠で十分です）。クレジットカードを登録しなくても無料枠の範囲で使えます。

---

### ステップ 1: Googleアカウントを用意する

GmailやYouTubeで使っているGoogleアカウントでOKです。
持っていない場合は [accounts.google.com](https://accounts.google.com) から作成してください。

---

### ステップ 2: Google AI Studio にアクセスする

1. ブラウザで **[https://aistudio.google.com](https://aistudio.google.com)** を開く
2. 右上の「**Sign in**」または「**ログイン**」をクリック
3. お使いのGoogleアカウントでログインする

---

### ステップ 3: APIキーを作成する

1. 左側のメニューから「**Get API key**」（APIキーを取得）をクリック
   ※ メニューが見当たらない場合は、左上のハンバーガーメニュー（三本線）をクリック

2. 「**Create API key**」（APIキーを作成）ボタンをクリック

3. 「**Create API key in new project**」を選択してクリック
   （「新しいプロジェクトにAPIキーを作成」という意味です）

4. しばらく待つと、英数字の長い文字列（例: `AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXX`）が表示される

5. 「**コピー**」ボタンをクリックしてAPIキーをコピーする
   ⚠️ **このキーは他人に見せないでください！**

---

### ステップ 4: アプリにAPIキーを設定する

1. Kitchen App を開く
2. 画面下のナビゲーションバーから「**設定**」（歯車アイコン）をタップ
3. 「**Gemini APIキー**」という項目を探す
4. 鍵アイコンをタップしてロックを解除する
5. テキストボックスにコピーしたAPIキーを貼り付ける
6. 「**保存**」ボタンをタップ
7. 「**接続テスト**」ボタンをタップして「接続成功」と表示されればOK！

---

### ステップ 5: 動作確認

1. 画面下のナビゲーションから「**AI解析**」を開く
2. 好きなレシピサイトのURLを貼り付けてみる
3. レシピが自動で読み込まれれば設定完了！

## 3. Googleログインの方法

### なぜGoogleログインが必要？

Googleカレンダーに献立を登録したり、家族とスケジュールを共有するには、
アプリが「あなたのGoogleカレンダーにアクセスする権限」をもらう必要があります。
その権限をもらうのが「Googleログイン」です。

> ℹ️ Googleログインには **Supabase**（スーパーベース）というサービスの設定も必要です。
> Supabaseとは、アプリのデータを管理するためのサービスです。[第5章](#5-supabaseのセットアップクラウド同期)を先に読んでから戻ってきてください。

---

### 前提条件

- Supabaseのセットアップが完了していること（第5章参照）
- Google Cloud Console でOAuthの設定が完了していること（以下で説明）

---

### ステップ 1: Google Cloud Console にアクセスする

**Google Cloud Console（グーグル・クラウド・コンソール）** は、GoogleのAPIを使うための管理画面です。
難しそうに聞こえますが、順番通りにやれば大丈夫です。

1. ブラウザで **[https://console.cloud.google.com](https://console.cloud.google.com)** を開く
2. Googleアカウントでログインする
3. 利用規約が表示されたら確認して「**同意する**」をクリック

---

### ステップ 2: プロジェクトを作成する

**プロジェクト** とは、アプリの設定をまとめる「フォルダ」のようなものです。

1. 画面上部の「**プロジェクトを選択**」をクリック
2. 右上の「**新しいプロジェクト**」をクリック
3. 以下を入力する：
   - プロジェクト名: `Kitchen App`（なんでもOK）
4. 「**作成**」をクリック
5. 作成されたプロジェクトを選択する

---

### ステップ 3: Google Calendar API を有効にする

1. 左側のメニューから「**APIとサービス**」→「**ライブラリ**」をクリック
2. 検索ボックスに「**Google Calendar API**」と入力して検索
3. 「Google Calendar API」をクリック
4. 「**有効にする**」ボタンをクリック
5. 「APIが有効になりました」と表示されればOK

---

### ステップ 4: OAuth同意画面を設定する

**OAuth同意画面** とは、「このアプリはあなたのGoogleカレンダーにアクセスしたい」という確認画面のことです。

1. 左側メニューから「**APIとサービス**」→「**OAuth同意画面**」をクリック
2. 「**外部**」を選択して「**作成**」をクリック
3. アプリ情報を入力する：
   - アプリ名: `Kitchen App`
   - ユーザーサポートメール: 自分のGmailアドレス
   - デベロッパーの連絡先: 自分のGmailアドレス
4. 「**保存して次へ**」をクリック
5. スコープの画面では「**保存して次へ**」をクリック（そのままでOK）
6. テストユーザーの画面で「**+ ADD USERS**」をクリックし、自分のGmailアドレスを追加
7. 「**保存して次へ**」→「**ダッシュボードに戻る**」をクリック

---

### ステップ 5: OAuth 2.0 クライアントIDを作成する

1. 左側メニューから「**APIとサービス**」→「**認証情報**」をクリック
2. 上部の「**+ 認証情報を作成**」→「**OAuth クライアント ID**」をクリック
3. アプリケーションの種類: 「**ウェブ アプリケーション**」を選択
4. 名前: `Kitchen App Web Client`（なんでもOK）
5. 「**承認済みのリダイレクト URI**」に以下を追加：
   ```
   https://あなたのプロジェクトID.supabase.co/auth/v1/callback
   ```
   ※ `あなたのプロジェクトID` はSupabaseのプロジェクトIDに置き換えてください
6. 「**作成**」をクリック
7. **クライアントID** と **クライアントシークレット** が表示される → メモしておく

---

### ステップ 6: SupabaseにGoogleプロバイダーを設定する

1. [app.supabase.com](https://app.supabase.com) を開いてログイン
2. 自分のプロジェクトを選択
3. 左側メニューから「**Authentication**」→「**Providers**」をクリック
4. 「**Google**」を見つけて「**Enable**」をオンにする
5. 以下を入力する：
   - **Client ID（クライアントID）**: ステップ5でメモしたクライアントID
   - **Client Secret（クライアントシークレット）**: ステップ5でメモしたクライアントシークレット
6. 「**Save**」をクリック

---

### ステップ 7: アプリでGoogleログインをする

1. Kitchen App を開く
2. 「**設定**」画面を開く
3. 画面上部のセクション一覧から「**アカウント**」を選ぶ
4. 「**Googleでログイン**」ボタンをタップ
5. Googleのログイン画面（`accounts.google.com`）が開く
6. 自分のGoogleアカウントを選択
7. 「カレンダーへのアクセスを許可しますか？」という確認が出たら「**許可**」をタップ
8. アプリに戻り、ログイン済みの表示になればOK！

> 補足: 現在、ホーム右上の「ログイン」ボタンは廃止されています。ログイン操作は設定画面の「アカウント」セクションから行ってください。

## 4. 家族カレンダーの設定

### この設定でできること

- 今日の献立をGoogleカレンダーに自動登録
- 家族のスケジュールを確認して、外食の日を避けた献立提案
- 食事時間をカスタマイズ（例：夕食は18:00〜19:00）

> 前提: [第3章のGoogleログイン](#3-googleログインの方法)が完了していること

---

### ステップ 1: Googleカレンダーで家族用カレンダーを作る

1. パソコンまたはスマートフォンで **[calendar.google.com](https://calendar.google.com)** を開く
2. 左側の「**他のカレンダー**」の横にある「**+**」ボタンをクリック
3. 「**新しいカレンダーを作成**」をクリック
4. 以下を入力する：
   - 名前: `家族の予定`（なんでもOK）
   - 説明: 任意
5. 「**カレンダーを作成**」をクリック

---

### ステップ 2: カレンダーIDを調べる

**カレンダーID** とは、それぞれのカレンダーを区別するための番号（ID）です。
メールアドレスのような見た目をしています（例: `xxxx@group.calendar.google.com`）。

1. Googleカレンダーの左側に、作ったカレンダーが表示されている
2. カレンダー名の右にある「**⋮**（三点メニュー）**」をクリック
3. 「**設定と共有**」をクリック
4. 下の方にスクロールして「**カレンダーの統合**」という項目を探す
5. 「**カレンダーID**」という欄に書かれた文字列をコピーする
   例: `abcdefg1234@group.calendar.google.com`

---

### ステップ 3: アプリにカレンダーIDを設定する

1. Kitchen App の「**設定**」を開く
2. 「**カレンダー設定**」という項目を探す
3. 以下を設定する：

| 設定項目 | 入力内容 | 説明 |
|---------|---------|------|
| 献立登録先カレンダー | コピーしたカレンダーID | 献立を登録するカレンダー |
| 家族カレンダー | コピーしたカレンダーID | 家族の予定を参照するカレンダー |
| 食事開始時間 | 例: `18:00` | 夕食の開始時間 |
| 食事終了時間 | 例: `19:00` | 夕食の終了時間 |

4. 「**保存**」をタップ

---

### ステップ 4: 動作確認

1. 「**週間献立**」画面を開く
2. 「**カレンダーに登録**」ボタンをタップ
3. Googleカレンダーを開いて、献立が登録されていればOK！

---

### 家族カレンダーとの連携について

家族カレンダーに「外食」「旅行」などの予定が入っている日は、
AIが献立を提案するときにその日を考慮してくれます。
うまく活用して、毎日の料理をもっとスムーズにしましょう！

## 5. Supabaseのセットアップ（クラウド同期）

### Supabaseとは？

**Supabase（スーパーベース）** は、データをインターネット上に保存・管理するためのサービスです。
料理に例えると、「冷蔵庫（スマホ本体）」のデータを「コンビニ（クラウド）」にもバックアップするイメージです。

#### Supabaseを設定するとできること：
- 複数のスマートフォンやパソコンでデータを同期
- スマホが壊れてもデータが消えない
- 家族でレシピや在庫を共有

#### 設定しなくてもできること：
- すべての基本機能はスマホ内だけで動作（インターネット不要）

> ⚠️ **費用について**: Supabaseには無料プランがあります（2025年現在、個人利用なら無料枠で十分です）

---

### ステップ 1: Supabaseのアカウントを作る

1. ブラウザで **[https://supabase.com](https://supabase.com)** を開く
2. 右上の「**Start your project**」または「**Sign Up**」をクリック
3. 「**Continue with GitHub**」（GitHubアカウントで登録）または「**Sign up with email**」（メールで登録）を選ぶ
   - GitHubアカウントを持っている場合はGitHubでの登録が楽です
   - メール登録の場合はメールアドレスとパスワードを入力
4. 確認メールが届いたらリンクをクリック（メール登録の場合）

---

### ステップ 2: プロジェクトを作る

1. ログイン後、「**New Project**」（新しいプロジェクト）をクリック
2. 以下を入力する：
   - 組織（Organization）: 「**New organization**」と表示されていればそのままでOK
   - プロジェクト名: `kitchen-app`（英語で入力）
   - データベースパスワード: 長くて覚えやすいパスワード（英数字混在推奨）
     ⚠️ メモしておいてください！後から変更が難しいです
   - リージョン（Region）: `Northeast Asia (Tokyo)` を選択（日本に近い方が速い）
3. 「**Create new project**」をクリック
4. 1〜2分待つとプロジェクトが作成される

---

### ステップ 3: APIキーとURLを確認する

1. プロジェクトのダッシュボードが開いたら、左側メニューの「**Project Settings**」をクリック
2. 「**API**」をクリック
3. 以下の2つをメモまたはコピーする：
   - **Project URL**: `https://xxxxxxxxxxxx.supabase.co` という形式
   - **anon public key**: 長い文字列（`eyJhbGci...`で始まる）

---

### ステップ 4: データベースのテーブルを作成する

アプリのデータを保存するための「テーブル（表）」をデータベースに作ります。

1. 左側メニューから「**SQL Editor**」をクリック
2. 「**+ New query**」をクリック
3. プロジェクト内の `supabase/migrations/001_initial_schema.sql` ファイルの中身をコピー
4. SQLエディタに貼り付けて「**Run**」（▶ ボタン）をクリック
5. 「**Success**」と表示されればOK
6. 同様に `supabase/migrations/002_rls_policies.sql` も実行する

---

### ステップ 5: アプリに設定を入力する

プロジェクトのルートフォルダに `.env` というファイルを作ります。

1. `.env.example` ファイルをコピーして `.env` という名前にする
   （ターミナルで `cp .env.example .env` と入力してもOK）

2. `.env` ファイルをテキストエディタで開き、以下を入力する：

```
VITE_SUPABASE_URL=https://あなたのプロジェクトID.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGci...（anon publicキー）
```

3. ファイルを保存する

4. アプリを再起動する（`npm run dev` を再度実行）

---

### ステップ 6: 動作確認

1. Kitchen App を開く
2. 「**設定**」画面を開く
3. 「**Googleでログイン**」でログインする
4. ログイン後、データがクラウドと同期されるようになります

## 6. アプリをホーム画面に追加する（PWAインストール）

Kitchen App は **PWA（プログレッシブ・ウェブ・アプリ）** に対応しています。
ホーム画面に追加すると、普通のアプリのように起動できて便利です！

### iPhoneの場合（Safari限定）

> ⚠️ **必ずSafariで開いてください。Chrome・Firefoxでは追加できません。**

1. **Safari** でアプリのURLを開く
2. 画面下部の「**共有ボタン**」（四角から矢印が出ているアイコン）をタップ
3. メニューを下にスクロールして「**ホーム画面に追加**」をタップ
4. アプリ名（「Kitchen App」）を確認して「**追加**」をタップ
5. ホーム画面にアイコンが追加される

---

### Androidの場合（Chrome）

1. **Chrome** でアプリのURLを開く
2. 画面上部または下部に「**アプリをインストール**」というバナーが表示されることがある
   表示された場合はそれをタップ
3. バナーが出ない場合：
   - Chrome の右上「**⋮（三点メニュー）**」をタップ
   - 「**アプリをインストール**」または「**ホーム画面に追加**」をタップ
4. 「**インストール**」をタップ
5. ホーム画面にアイコンが追加される

---

### オフライン利用について

一度アプリを開いてデータを読み込んだあとは、
インターネットに繋がっていなくてもレシピの閲覧・在庫管理などができます。
（AI機能や カレンダー連携はオンライン必須）

## 7. トラブルシューティング

### Q: Gemini APIキーを入力したのに「接続失敗」と表示される

**確認ポイント:**
1. キーの先頭・末尾にスペースが入っていないか確認する
2. Google AI Studio で作成したキーが正しくコピーできているか確認する（全部で39文字前後）
3. Google AI Studio の無料枠の上限に達していないか確認する（[aistudio.google.com](https://aistudio.google.com) でステータスを確認）

---

### Q: Googleログインができない・エラーが出る

**確認ポイント:**
1. 設定画面の「アカウント」セクションに「Googleでログイン」ボタンが表示されているか確認する
2. `VITE_GOOGLE_CLIENT_ID` が環境変数に設定されているか確認する（未設定だとログインボタンは無効化されます）
3. Supabaseの設定でGoogle プロバイダーが「有効」になっているか確認する
4. Google Cloud Console で OAuth同意画面の「テストユーザー」に自分のGmailを追加したか確認する
5. リダイレクトURIが正確に入力されているか確認する（スラッシュの有無に注意）
6. アプリ内ブラウザではなく、通常のChrome/Safariで開いて再試行する（ポップアップ制限回避）

> `https://accounts.google.com` を直接開くこと自体は可能ですが、これは「Googleアカウントへのログイン」です。Kitchen App 連携を完了するには、必ずアプリの「Googleでログイン」ボタンからOAuthフローを開始してください。

---

### Q: カレンダーIDを設定したのに献立が登録されない

**確認ポイント:**
1. Googleログインが完了しているか確認する
2. カレンダーIDの末尾に余分なスペースや改行が入っていないか確認する
3. Googleカレンダーを開いて、そのカレンダーにアクセス権があるか確認する

---

### Q: iPhoneでホーム画面に追加できない

**確認ポイント:**
1. 必ず **Safari** で開いているか確認する（ChromeやFirefoxでは追加できない）
2. iOS のバージョンが 16.4 以上か確認する（古いバージョンはPWAに対応していない場合がある）

---

### Q: アプリのデータが消えてしまった

データはスマホ本体に保存されています。以下の操作でデータが消える場合があります：

- ブラウザのキャッシュをすべて削除した
- プライベートブラウジング（シークレットモード）で使用していた
- アプリをアンインストールした

**対策:** [第5章のSupabase設定](#5-supabaseのセットアップクラウド同期)を行うと、クラウドにバックアップが作られて安心です。

### Q: `.env` ファイルはどこに作ればいい？

アプリのフォルダの一番上（ルートディレクトリ）に作ります。
`package.json` と同じ場所に置けばOKです。

```
my-recipe-app/
├── .env          ← ここに作る！
├── .env.example  ← これをコピーして使う
├── package.json
├── src/
└── ...
```
